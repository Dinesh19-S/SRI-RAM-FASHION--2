effectively manage their finances by tracking their income, expenses, and savings. It

utility. For a user -friendly experience, the system can support both web and mobile

include recu rring transaction management, AI -driven spending suggestions, and currency

2.1 IDENTIFICATION OF NEED  
                    The need for a Money Tracker System arises from the increasing 
challenges individuals and businesses face in managing their finances effectively. 
Financial mismanagement is a common issue, with many struggling to maintain accurate 
records of income and expenses. This often leads to a lack of awareness of spending 
patterns, making it difficult to identify unnecessary expenses or trends in overspending. 
Traditional methods like spreadsheets or notebooks are time -consuming, error -prone, and 
lack the real -time capabilities required for acc urate financial tracking . 
Additionally, creating and adhering to budgets is a challenge without proper tools 
to allocate funds and monitor progress effectively. Many users also lack the means to set 
and achieve financial goals, such as saving for emergenci es or planning for future 
investments. Existing tools often fall short in providing advanced insights, such as cash 
flow forecasting or category -specific analysis, leaving users with limited understanding 
of their financial health.  
Security is another crit ical concern, as users are hesitant to trust systems that do 
not prioritize data protection. Moreover, families, roommates, or business partners often 
face difficulties managing shared finances due to the lack of collaborative features in 
current solutions . Accessibility issues, such as platform dependency or lack of offline 
functionality, further limit the usability of existing tools.  
Given these challenges, there is a clear need for a secure, user -friendly, and 
feature -rich Money Tracker System that provi des real -time tracking, advanced analytics, 
budgeting tools, financial goal tracking, and collaborative capabilities. Such a system can 
empower users to take control of their finances, make informed decisions, and achieve 
their financial goals efficiently.

Data Independence  
Data independence is a core feature o f the Budget Tracker System, ensuring that 
modifications in the database do not disrupt its functionality. With Supabase  and its

maintaining flexibility in managing fina ncial data, allowing changes to the database

 Physical Data Independence : The underlying storage mechanisms, such as 
indexing and optimization of tables, can be adjusted or modified without affecting 
the user experience or functionality of the app. Users will continue to interact with 
the system seamlessly, regardless of the changes made to the physical storage.  
 Logical Data Independence : The system ’s schema can be updated, such as 
adding new fields (e.g., additional categories for income or expenses) or 
modifying relationships between data tables (e.g., linking users to custom 
financial goals), without disrupting the workflow or existing features of the 
application.

4.3 ERROR HANDLING  
                         Error handling is a critical part of the Budget Tracker System  to ensure 
that the application runs smoothly and provides a seamless experience for users. The 
system employs robust error handling strategies both on the frontend and backend to 
catch, manage, and display errors in a user -friendly manner.  
 
1. FrontEnd : 
On the frontend, error handling is integrated within the user interface to manage 
potential issues such as invalid input, network failures, or authentication errors. 
For exa mple:  
o Form Validation : User inputs such as income or expense amounts are 
validated to ensure they are in the correct format and fall within acceptable 
ranges. Error messages are shown directly on the form if the user enters 
invalid data (e.g., non -numeric characters).  
o Network Errors : In case of network failures while fetching or sending 
data, users receive a clear error message, informing them of the issue and 
prompting them to retry.  
o Authentication Errors : If login or sign -up fails (due to incorrect 
creden tials or network issues), users are notified with an appropriate 
message guiding them to correct the issue.  
 
 
 
 
 
 

--- page 23 ---
23 
 
2. Backend : 
On the backend, error handling is crucial for managing issues related to data 
retrieval, user requests, and API calls. The system use s: 
o Try-Catch Blocks : For asynchronous operations such as database queries, 
try-catch blocks are used to catch errors and respond with meaningful 
messages.  
o HTTP Status Codes : The backend uses appropriate HTTP status codes 
(e.g., 400 for bad requests, 500 f or server errors) to communicate the type 
of error to the frontend, allowing it to display relevant messages to the 
user. 
o Custom Error Handling Middleware : Express.js middleware is used to 
catch any unhandled errors and respond with generic error messages,  
ensuring that users are not exposed to internal system details. This 
prevents the app from crashing and ensures a smooth user experience.  
 
3. Database :  
Error handling is also integrated at the database level to catch issues related to 
database queries, such as failed insertions, updates, or invalid data entries. Supabase's 
API will return clear error responses that the backend processes to notify users of  any 
issues with their data, such as "Unable to add income entry due to invalid data."
